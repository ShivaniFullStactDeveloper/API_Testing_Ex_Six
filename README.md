# SCOS API Testing Exercise Six

## Project Overview

This project contains API testing implementation for backend services using automated and manual testing approaches. The purpose of this exercise is to learn and validate REST API functionality, response handling, authentication, and error validation.

The project focuses on verifying API behavior, request/response validation, authentication flow, status code handling, and backend functionality testing.
 
## Exercise Objective
 
- Understand API testing concepts
- Validate REST API endpoints
- Perform request and response validation
- Test authentication and authorization
- Verify status codes and error handling
- Practice backend testing workflows
 
## Technologies Used
 
- JavaScript
- Node.js
- Jest
- Postman
- REST APIs
- JSON
- Express.js
 
## Project Structure
 
src/ → Backend source code  
tests/ → API test files  
auth/ → Authentication APIs  
user/ → User module APIs  
config/ → Environment configuration  
 
## Implemented Test Scenarios
 
- User registration API testing

- Authentication validation
- Invalid credential validation

- Duplicate user validation

- Status code verification
- API response validation
- CRUD operation testing
- Security testing


